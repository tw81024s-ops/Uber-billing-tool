# UBER 帳單整理工具

一個放在瀏覽器裡跑的 UBER 帳單整理工具。匯入帳單檔，自動依「商家訂單 ID」合併、加總金額；再匯入貨故請賠檔，自動產出貨故請賠與運費回溯的調整明細。產出 Excel 直接下載。

**所有資料在你的瀏覽器內處理，不會上傳到任何伺服器。**

## 線上使用

把這個 repo 開啟 GitHub Pages 後，直接打開 `https://你的帳號.github.io/uber-billing-tool/` 就能用。

或下載整個資料夾到本機，用瀏覽器打開 `index.html` 也可以。

## 功能

**Step 1 - 上傳原始帳單**
拖檔或點選 .xlsx／.xls／.csv。工具會智慧判讀欄位（商家訂單 ID、帳單金額），自動猜對了就直接用，猜錯可以手動修正。按「預覽合併結果」，依商家訂單 ID 合併、金額加總。會顯示原始筆數、合併後筆數、總額是否一致。

**Step 2 - 上傳貨故請賠檔（可選）**
工具會：
- 把同查貨號碼的「扣款金額（含稅）」加總、轉負數，產出「貨故請賠」一列。
- 用主檔該單號金額（絕對值）減掉貨故加總，差額是 41 或 45 時，自動產出「貨故運費回溯」一列（負數）。差額為 0 不產生；差額對不上就放到「待人工確認」分頁。

**Step 3 - 上傳虛扣／重複收款檔（可選）**
從「重複收款 Summary」分頁讀取虛扣單號與應退金額，放到「待人工確認」分頁，金額拆分由使用者自行決定。

**Step 4 - 產出**
產出 Excel 包含兩個分頁：
- **Final**：左塊是合併後帳單（商家訂單 ID／加總帳單金額），右塊是調整明細（單號／金額／備註）。
- **待人工確認**：工具抓不到、需要人工查詢追蹤的單號（差額異常、主檔查無、重複收款應退拆分等）。

工具不會猜「實際派單失敗、信件通知加入請款、UBER錯址、1月已請款、momo尚未銷單」這類備註——它們所需的判斷依據不在原始檔裡，會放到待人工確認分頁讓使用者自行填寫。

## 核心規則摘要

| 項目 | 規則 |
|---|---|
| 主檔合併 | 商家訂單 ID 分組，帳單金額加總 |
| 貨故請賠 | 同查貨號碼的「含稅金額」加總、轉負數 |
| 貨故運費回溯 | 主檔金額（絕對值）− 貨故含稅加總，差額為 41/45 時自動產出，為 0 不產出 |
| 異常 | 差額非運費（41/45），標記到待人工確認分頁 |
| 重複收款 | 從 Summary 讀應退金額，列入待人工確認，拆分由人工決定 |

## 智慧判讀欄位

工具會用兩種訊號判斷哪一欄是什麼：
- **欄位名稱關鍵字**：例如「商家訂單ID」「external_order_id」「查貨號碼」「虛扣單號」都會被認成單號欄位；「帳單金額」「amount」「含稅」會被認成金額欄位。
- **資料樣本特徵**：訂單號通常是 10-15 位數字串、金額通常是純數字、Uber 訂單 ID 通常是 `#xxxxx` 短碼。

判讀對了會顯示「已自動判讀」標籤，使用者可以隨時手動修正。

## 技術說明

完全純前端，沒有後端、不需要安裝任何東西。

- HTML / CSS / 原生 JavaScript
- 透過 [SheetJS](https://github.com/SheetJS/sheetjs) (CDN) 讀寫 Excel
- 字體：Noto Sans TC + Plus Jakarta Sans (Google Fonts)

支援的瀏覽器：Chrome／Edge／Firefox／Safari 近兩年版本。

## 部署到 GitHub Pages

1. Fork 或下載此 repo，上傳到你的 GitHub。
2. 進到 repo 的 Settings → Pages。
3. Source 選 `Deploy from a branch`，Branch 選 `main` 或 `master`，資料夾選 `/ (root)`。
4. 等 1～2 分鐘，就會有 `https://你的帳號.github.io/uber-billing-tool/` 可以用。
